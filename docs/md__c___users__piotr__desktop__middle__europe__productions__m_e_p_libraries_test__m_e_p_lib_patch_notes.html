1.01.2021

MEP lib version alpha 1.3

-Added OSSetUp.h which specifies the operating systems. Currently only MEP_WINDOWS is avalible.
-Added WindowData.h responsible for window and view management.
-Changed naming WindowsStat.h -> WindowStat.h
-Reorganized resource management:
*Now there is a Assets.h which defines some basic assets.
*Now resource have its own unique U_int32(MEPGraphics/Config.h) ID.
*The ID needs to be specified on the input.
*The ID needs to be unique in every group.
*Group is defined by the user. Currently there are 2 "hard coded" groups.
1.UserAssets - a default assets direction. If you do not specify the group your resource goes here.
2.HUB - all of the HUB assets.
*To access the resource u do need to use one of the methods:
getObject(const U_int32 ID, const U_int32 group = -1)
getObject(const std::string& name, const U_int32 group = -1)
*To access the build in assets/your simply:
getObject(MEP::HUB::Box, MEP::AssetsGroup::HUB)
*Hint: To enhance your work create enums same as in the Assets.h
Do not use AssetsGroup (-1, -2) there are reserved for the engine.
-The 1.3 version comes with the modified version of SFML 2.5.1
*In order to achive full mep experience u will need to download SFML 2.5.1 and replace the files.
*All of the files are in the SFML_2.5.1_Changes folder.
*Please keep in mind that I do not own any rights the the files.
*All of the changes are marked in the following way:
/// <MEP Modification>
Implementation
/// </MEP>
*Those are the changes related to sf::Style::None not really looking the best.
A new *Windows exclusive styles are avalible. sf::Style::Modern and sf::Style::Modern_Resize == sf::Style::Modern | sf::Style::Resize.
*If you do not edit the SFML sf::Style::None is the equivalent of sf::Style::Modern.
*Unfortuantely, resizing the sf::Style::None is not avalible in the SFML.
*What is the difference between None and Modern?
*Modern style has the WS_POPUP tag (borderless window) but with the addition of windows animations and drop shadow.
*Furthermore it is possible to resize it thanks to the sf::Style::Modern_Resize.
-A new Drawable tag & more. *It is possible to mix the drawable tags (they are bitwise now).
*A new tag Resize_Pos was added.
*When set it is going to be used when the window is resized.
*It executes the calculation method to recalculate the position of an objects.
*You do need to provide the method with the use of: addMethodPos(...)
